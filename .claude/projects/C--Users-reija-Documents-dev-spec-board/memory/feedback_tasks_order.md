---
name: tasks-reverse-order
description: Tasks.md should show next/in-progress work at the top, completed phases at the bottom. Reverse chronological.
type: feedback
---

Tasks.md phases should be ordered with the NEXT work at the top and completed phases at the bottom.

**Why:** For large projects with many phases, the user has to scroll past all completed work to find what's currently being done. The active/next phase should always be immediately visible.

**How to apply:** When writing or updating tasks.md, put the current/next phase at the top. Completed phases go below in reverse order. Collapse completed phases to summary lines when they get long.
