During /speckit.implement, update tasks.md after EACH PHASE completes — not in a batch at the end.

**Why:** The user has zero visibility into what's happening if tasks are only marked done at the very end. The whole point of tasks.md is progress tracking. Batching defeats the purpose.

**How to apply:** After completing each phase during implement, immediately edit tasks.md to check off the completed tasks. This way if the user checks tasks.md mid-implementation, they see accurate progress.
